About Me

I’m Full Stack Developer or Frontend Developer with 5+ years of experience. In the past I have also worked as a developer with Geographic Information Systems (GIS). I’m good with data-driven applications and helping organizations collect, store, share and visualize their data. I also like creating engaging web experiences.

From 2019 to 2025 I was working with 3vGeomatics to build their web platform for analyzing InSAR data and ground movement called Motionary. The application was designed as an interactive map to with rich geospatial and data visualization and export tools. I helped turn the application from an initial prototype into a full-featured application relied upon by dozens of businesses. Today Motionary provides access to hundreds of gigabytes of spatial and non-spatial data created and hosted by 3vGeomatics. The teams supporting it has grown to several frontend and backend/API developers.

Although I primarily worked in frontend space, I’ve contributed to and supported all the application code and infrastructure over the years. For example, I’ve built tools to monitor the application up-time, set up logging and metric collection in AWS for infrastructure, designed load tests for capacity planning and chased down data issues within AWS, ArcGIS Server, PostGreSQL (PostGIS) and docker containerized aps as part of my regular work.

Towards the last years I was focused on integrating new API sources of data into our frontend map application. I worked with backend developers to set up a custom Vector API service to provide data for our app as well as with our Raster API team to incorporate dynamic time-based Raster information.

Previous to my work at 3vGeomatics, I worked as a GIS Developer on contract with the province of Nova Scotia, mainly working with ESRI’s ArcObjects SDK, C# and .NET solutions helping their emergency services apps. Before that I was a Data Analyst with Environment and Climate Change Canada working with ESRI ArcGIS desktop tools and a fair bit of Python code.

Way back a long time ago, before going back to school, I was a GIS Analyst. I worked for several years at Stantec Consulting in Halifax, Nova Scotia and around Atlantic Canada. I supported their environmental teams, mainly doing a lot of data wrangling, mobile GPS support and cartographic production work. It was how I got into coding.

I like building UX experiences that make data and information easy to understand. I enjoy my craft as a developer and setting up tools and processes that allow teams to get things done and work together better. I am constantly learning new things and being curious. I’ve had the privilege to work with a lot of really smart, highly educated people in various roles and pride myself on learning about what they do enough to understand and champion their work.

I enjoy challenges and opportunities to solve technical problems, research solutions and integrate new technologies. Some of my skills:

  • Modern frontend web development: particularly React/Redux, some Vue/VueX,
  • MV* web frameworks and content management systems: particularly python-based Django, Flask
  • Geospatial development: mainly ESRI ArcGIS Map SDK for JavaScript, ESRI ArcObjects, Opensource GDAL
  • Python scripting development
  • Version control systems (Git) and hosted Repositories (Gitlab)
  • Database design and development: PostGreSQL
  • Windows and Linux administration and scripting: Bash
  • Needs assessments, requirements gathering, issue tracking, and documentation (Gitlab Issues, Jira, Confluence)
  • Data management and transformation (Relational, Dimensional Data, Structured and Unstructured Data, JSON, XML, etc.)
  • Docker containerization primarily for development workflows
